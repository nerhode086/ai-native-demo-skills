# SettlementSystem

- 适用场景：战斗、Roguelike run、竞技场、副本、排位结算。
- 常见触发词：结算、胜负、分数、通关、退出 run、战斗结果。
- 典型输入：战斗结果、分数、耗时、掉落、玩家状态、服务端验证信息。
- 典型输出：结算结果、奖励、进度、排行分、日志。
- 配置关注：结算规则、奖励、评分、失败处理、反作弊。
- 协议关注：提交结算、结算响应、错误码、重复提交、断线恢复。
- QA 关注：重复结算、离线结算、异常退出、奖励发放、排行更新。
- 常见组合：`RewardSystem`、`RankingSystem`、`DropSystem`、`TaskCondition`。
- 不适用场景：无最终结果确认的普通 UI 操作。
- 人工 Review：战斗结算、奖励、排行和反作弊链路必须 Review。
