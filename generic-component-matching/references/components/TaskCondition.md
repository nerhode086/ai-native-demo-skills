# TaskCondition

- 适用场景：任务进度、成就、解锁条件、引导步骤、活动目标。
- 常见触发词：任务、进度、完成、条件、成就、目标、引导。
- 典型输入：事件类型、条件 ID、目标值、当前进度、玩家状态。
- 典型输出：进度 delta、完成状态、奖励可领取状态、客户端提示。
- 配置关注：条件类型、目标值、事件映射、重置规则、活动周期。
- 协议关注：进度同步、完成状态、错误码、断线恢复。
- QA 关注：重复事件、进度溢出、重置、配置缺失、客户端显示一致性。
- 常见组合：`InteractionObject`、`RewardSystem`、`LevelUp`、`ActivitySystem`。
- 不适用场景：没有目标进度或完成条件的纯奖励。
- 人工 Review：任务进度绑定奖励或活动结算时需要 Review。
